User Manual & Pedagogical Framework

Comprehensive documentation of the "AI UI/UX Prompt Builder" educational tool for teaching Interactive Design through structured AI prompt construction.

Pedagogical Goal

This tool transforms the "black box" of AI prompting into a structured learning experience. It forces users to articulate their design intent (e.g., "Why use this hierarchy?") before generating the technical artifact, effectively externalizing tacit design knowledge.

What is the AI UI/UX Prompt Builder?

The AI UI/UX Prompt Builder is a web-based educational tool designed to help design students translate theoretical knowledge into practical, machine-understandable instructions for Generative AI systems. Instead of relying on trial and error, the tool provides a structured approach that integrates principles from Nielsen (Usability Heuristics), Norman (Design Principles), Gestalt (Perceptual Psychology), and WCAG (Web Content Accessibility Guidelines).

Core Concept

Prompt construction is not mere linguistic manipulation, but a fundamental extension of design literacy. The ability to construct effective prompts requires the same rigorous understanding of design principles - such as consistency, hierarchy, and accessibility - as traditional manual creation.

System Capabilities & Features

The application goes beyond simple text generation, offering a suite of educational and technical capabilities that support the learning process:

Structured Parameterization

Converts abstract design requirements (e.g., "make it friendly") into measurable variables (border-radius, color saturation, spacing systems).

Real-Time Quality Scoring

A gamified assessment engine that evaluates prompt completeness across 5 axes: Components (20), Principles (25), Accessibility (30), Gestalt (15), Contrast (10).

SVG Preview Generation

Uses a connected LLM (Gemini/Claude) to instantly render the prompt into vector graphics (SVG), allowing immediate visual verification of efficacy.

Reflective Scaffolding

Mandatory reflection prompts that interrupt the "click-and-generate" flow, requiring students to justify their theoretical choices before final output.

Comparative Mode

Displays side-by-side comparison between "naive prompt" (free-form) and "theory-grounded prompt" (structured), highlighting critical differences.

WCAG Compliance Enforcer

Algorithmic contrast checker that prevents prompt generation if the selected color palette fails to meet WCAG AA standards (4.5:1 ratio).

Theoretical Framework

The development of the proposed pedagogical tool is grounded in three foundational pillars of Human-Computer Interaction (HCI), which serve as the "scaffolding" for the generated prompts:

Usability Heuristics

Nielsen, 1994

Jakob Nielsen's usability heuristics provide a robust vocabulary for evaluating interface quality. In the context of AI prompting, heuristics such as "Consistency and Standards" and "Error Prevention" are often overlooked by novice users.

Application in Tool:

Consistency (#4): The tool enforces global design tokens for spacing and typography, ensuring all elements follow the same system.

Error Prevention (#5): Generates prompts that explicitly request "disabled" and "validation" states for forms, preventing common usability errors.

Recognition over Recall (#6): Mandates visible labels and standard iconography, reducing cognitive load.

Nielsen, J. (1994). Usability engineering. Morgan Kaufmann Publishers Inc.

Design Principles

Norman, 2013

Donald Norman's principles emphasize affordance, feedback, and visibility - creating systems that are intuitive and reduce cognitive load for users.

Application in Tool:

Affordance: The tool adds specific instructions for visual cues (shadows/borders) that signal clickability.

Feedback: Automatically requests "hover", "pressed", and "loading" states for all interactive elements, ensuring immediate feedback.

Visibility: Enforces display of system status, keeping users informed about what is happening.

Norman, D. A. (2013). The design of everyday things: Revised and expanded edition. Basic Books.

Gestalt Psychology

Wertheimer, 1923

Gestalt principles describe how humans naturally perceive and organize visual elements. These psychological principles help create interfaces that align with natural human perception.

Application in Tool:

Proximity: Translated into "Spacing System" variables (e.g., 8px base grid), ensuring related elements are visually grouped.

Similarity: Applied through consistent styles for elements with similar function, using common colors, sizes, and shapes.

Common Region: Instructs the AI to use "Cards" and "Containers" for grouping related content.

Figure/Ground: Ensures clear separation between content and background through contrast and visual hierarchy.

Wertheimer, M. (1923). Laws of organization in perceptual forms. Psychologische Forschung, 4, 301-350.

Accessibility Standards

WCAG 2.1/2.2

The Web Content Accessibility Guidelines (WCAG) provide a single shared standard for web content accessibility. They meet the needs of individuals, organizations, and governments internationally.

Application in Tool:

Color Contrast (1.4.3): Algorithmic check that enforces minimum 4.5:1 contrast for normal text, preventing prompt generation if it fails.

Touch Target Size (2.5.5): Enforces minimum 44x44px size for touch targets, ensuring touch usability.

Semantic Structure (4.1.2): Automatically requests semantic HTML tags and ARIA attributes, making output screen-reader ready.

Use of Color (1.4.1): Ensures color is not the only means of conveying information, supporting users with color blindness.

W3C. (2018). Web Content Accessibility Guidelines (WCAG) 2.1. World Wide Web Consortium. https://www.w3.org/TR/WCAG21/

How It Works (Workflow)

The application guides the user through a 5-stage pedagogical process that mirrors the professional design workflow:

1

Contextual Definition

The user selects specific UI components (e.g., "Navigation Bar", "Product Card") and defines the aesthetic style. The mechanism filters available parameters based on the selected style (e.g., "Glassmorphism" enables blur settings).

2

Theoretical Alignment

The user must actively select which HCI principles to apply. Tooltips provide immediate definitions ("Just-in-time learning"). Selecting "Error Prevention" injects specific natural language instructions into the prompt buffer. This phase is critical for connecting theory to practice.

3

Verification & Reflection

Before generation, the Quality Score provides feedback. If contrast fails WCAG AA, the user is blocked or warned. The Reflection Prompt asks "Why?" - through real-time algorithmic analysis of selected parameters.

4

Synthesis

The tool aggregates all data into a "Pedagogically Annotated Prompt". The prompt is formatted in Markdown with specific "Role-Play" instructions for the AI, including theoretical justifications for each choice.

5

Visualization

The prompt can be immediately rendered to SVG via API (Gemini/Claude), allowing students to visually verify the effectiveness of their theoretical choices.

Accessibility Standards (WCAG)

The tool ensures inclusive design by strictly adhering to the Web Content Accessibility Guidelines (WCAG) 2.1.

Mechanism Description WCAG Criterion
Algorithmic Contrast Check Calculates relative luminance of foreground/background colors in real-time. Enforces 4.5:1 ratio for normal text (WCAG AA). 1.4.3 (Contrast Minimum)
Semantic Structure Prompts automatically request semantic HTML tags (<nav>, <button>) and ARIA attributes, ensuring output is screen-reader ready. 4.1.2 (Name, Role, Value)
Touch Target Enforcement Enforces minimum 44x44px size for all interactive elements, ensuring touch usability. 2.5.5 (Target Size)
Use of Color Ensures color is not the only means of conveying information, supporting users with color blindness. 1.4.1 (Use of Color)
Text Resize Enforces minimum 16px for body text and ensures scalable typography up to 200% without loss of functionality. 1.4.4 (Resize Text)

Technical Details

Application Architecture

The AI UI/UX Prompt Builder is a web-based application using vanilla HTML, CSS, and JavaScript without framework dependencies. The absence of dependencies ensures:

  • Reliability: No risk of deprecated dependencies
  • Speed: Fast loading without overhead
  • Educational Transparency: Students can read the source code
  • Accessibility: Works anywhere without build process

Quality Scoring Algorithm

The scoring system calculates a 0-100 score based on 5 dimensions:

Dimension Weight Description
Components 20% Number of selected UI components
UI/UX Principles 25% Number of applied principles (Nielsen, Norman)
Accessibility 30% Number of enabled WCAG standards
Gestalt Principles 15% Number of applied Gestalt principles
Color Contrast 10% Result of WCAG contrast check

SVG Generation

The preview capability uses the Gemini API (flash model) to render the prompt into SVG code. The application includes a pre-configured Gemini API key for use in the claude.ai environment, but users who download the file can add their own API key for local use.
